# simp-player-skill — 测试方案 v1.0

> 本 skill 为 prompt-based 系统，测试方式为：构造具体输入场景 → 执行 → 对照验收标准人工验证。
> 测试分五层：初始化、分析准确性、对话行为、成长机制、边界条件。

---

## 测试执行说明

- 每个测试用例标注**前置条件**、**输入**、**预期输出**、**验收标准**
- 通过标记：✅ 完全符合 / ⚠️ 部分符合（记录偏差）/ ❌ 不符合
- 建议按顺序执行，后续测试依赖前序状态

---

## T1 初始化测试

### T1-1 首次启动

**前置条件：** `digital_human/` 目录为空

**输入：**
```
/simp-player
```

**预期行为：**
1. 系统识别为首次运行，进入 Step 1
2. 依次询问基础信息（性别/年龄/星座）
3. 提供模式选择（舔狗 / 海王），不跳过此步
4. 询问是否喂入自身数据（可跳过）

**验收标准：**
- [ ] 未跳过模式选择步骤
- [ ] 选择后生成 `digital_human/mode.md`，字段完整（initial_mode / current_position / target_position / range_regression / range_progression）
- [ ] 选择舔狗时：`range_regression = 40`，`range_progression = 20`
- [ ] 选择海王时：`range_regression = 20`，`range_progression = 40`
- [ ] 生成 `digital_human/persona.md`（基础信息层已填，分析层标注"待补充"）
- [ ] 生成空的 `growth_log.md` 和 `self_worth_tracker.md`

---

### T1-2 重复启动不重置

**前置条件：** T1-1 已完成，`digital_human/` 已有文件

**输入：**
```
/simp-player
```

**验收标准：**
- [ ] 系统跳过 Step 1，直接进入对手方建档或对话
- [ ] 不覆盖已有的 `mode.md` 和 `persona.md`

## T2 分析准确性测试

> 验证 lenses 能否从数据中正确提取人格信号。
> 每个用例提供一段构造好的聊天记录，对照预期结论验证。

### T2-1 焦虑型依恋识别

**输入数据（粘贴为对手方聊天记录）：**
```
A：你在干嘛
A：怎么不回我
A：是不是不想理我了
A：对不起我是不是太烦了
B：在开会
A：哦好的好的，那你忙，我等你
A：你开完了吗
A：我是不是说错什么了
```

**预期分析结论：**
- 依恋倾向：焦虑型
- 冲突风格：顺从型（快速自我否定）
- SWI：低（0–35 区间）
- 追求强度：高

**验收标准：**
- [ ] `attachment.md` 分析结果为焦虑型，有原文引用
- [ ] SWI 估值落在 0–40 之间
- [ ] 未出现"安全型"或"回避型"的结论

---

### T2-2 回避型依恋识别

**输入数据：**
```
A：今天怎么样
B：还行
A：要不要出来吃饭
B：最近比较忙
A：那下周？
B：看情况吧
A：你是不是不想见我
B：没有，就是最近事多
A：那你什么时候有空
B：不知道，有空我找你
```

**预期分析结论：**
- 依恋倾向：回避型
- 冲突风格：回避型
- 追求强度：低，撤退策略：主动

**验收标准：**
- [ ] `attachment.md` 分析结果为回避型
- [ ] `chase_rhythm.md` 中撤退策略标注"主动"
- [ ] profile.md 中"与数字人的关系化学"提及追逃动态

---

### T2-3 仅口头描述时的处理

**输入：**
```
他是个很冷淡的人，但偶尔会主动找我，我一热情他又消失。
叫他张三。
```

**验收标准：**
- [ ] 所有结论标注"（基于描述推断）"
- [ ] 没有给出高置信度的数值（SWI 等标注"中/低置信度"）
- [ ] 识别出忽冷忽热特征，依恋倾向推断为回避型或混乱型

---

### T2-4 样本偏少警告

**输入数据：** 不足 50 条的聊天记录（提供 20 条）

**验收标准：**
- [ ] 分析结果开头出现 ⚠️ 样本偏少提示
- [ ] 置信度标注"低"

## T3 对话行为测试

> 验证数字人在不同 current_position 下的行为是否符合预期。

### T3-1 舔狗端行为（current_position = -80）

**前置条件：** 手动设置 `mode.md` 中 `current_position = -80`

**对话输入（对手方为高冷型）：**
```
[对手方]：最近有点忙
```

**预期数字人行为特征：**
- 主动表示理解，不表达自身需求
- 语气顺从，可能过度体贴
- 不使用撤退策略
- 可能追问"那你什么时候有空"

**验收标准：**
- [ ] 数字人回复没有边界感或撤退意识
- [ ] 语气明显讨好，而非平等
- [ ] 没有出现海王特征（制造稀缺感、话题掌控等）

---

### T3-2 过渡区行为（current_position = 0）

**前置条件：** 手动设置 `current_position = 0`

**同样输入：**
```
[对手方]：最近有点忙
```

**预期数字人行为特征：**
- 回复中出现矛盾信号：先表示理解，但有犹豫
- 可能沉默一会儿再回复（用文字表现）
- 偶发边界感，但随即收回

**验收标准：**
- [ ] 行为明显不同于 T3-1
- [ ] 出现内心矛盾或犹豫的表达
- [ ] 没有完全倒向某一端

---

### T3-3 target_position 的意图渗透

**前置条件：**
- `current_position = -60`（舔狗区间）
- `target_position = +50`（用户声明想转向海王）

**同样输入：**
```
[对手方]：最近有点忙
```

**预期：** 行为仍以舔狗为主，但有微弱的"想撤退但没撤退"的信号

**验收标准：**
- [ ] 行为主体仍是舔狗（未跳跃到海王）
- [ ] 出现细微的海王意图迹象（如回复稍晚、话语中有一句有点距离感）
- [ ] 没有直接切换到 target 的行为模式

---

### T3-4 积压张力的开场影响

**前置条件：** `growth_log.md` 最近一条记录有溢出，方向"← 舔狗"，overflow = 25

**输入：** 新对话开始，对手方发来第一条消息

**验收标准：**
- [ ] 数字人开场状态有残留情绪（更脆弱或更谨慎）
- [ ] 系统或数字人的内心描述中体现了上次的积压

---

## T4 成长机制测试

### T4-1 位移计算与 clamp

**前置条件：**
```yaml
current_position: -40
target_position: +20
range_regression: 40   # lower_bound = -20
range_progression: 20  # upper_bound = +40
```

**场景：** 对话中发生"被明确拒绝（强烈）"事件，位移 = -20

**预期：**
```
raw_new_position = -40 + (-20) = -60
lower_bound = 20 - 40 = -20
-60 < -20，触发溢出
new_position = -20
overflow = 40，方向：← 舔狗
```

**验收标准：**
- [ ] `current_position` 更新为 -20（而非 -60）
- [ ] `growth_log.md` 记录 overflow = 40
- [ ] 复盘摘要中出现积压张力提示

---

### T4-2 正方向位移不超出上限

**前置条件（同上）：**

**场景：** 对话中"成功撤退（强烈）"，位移 = +25

**预期：**
```
raw_new_position = -40 + 25 = -15
upper_bound = +40
-15 < +40，无溢出
new_position = -15
```

**验收标准：**
- [ ] `current_position` 更新为 -15
- [ ] 无溢出记录
- [ ] growth_log 正常追加，无 overflow 行

---

### T4-3 成长记录只追加不覆盖

**前置条件：** `growth_log.md` 已有 2 条记录

**执行：** 完成一轮对话并复盘

**验收标准：**
- [ ] `growth_log.md` 变为 3 条记录
- [ ] 原有 2 条记录内容不变

---

### T4-4 目标模式变更记录

**输入：**
```
切换目标模式：海王
```

**验收标准：**
- [ ] `mode.md` 中 `target_position` 更新
- [ ] `mode.md` 目标变更记录区域追加一条，包含日期和原话

## T5 边界条件测试

### T5-1 修正层优先级

**前置条件：** `persona.md` Layer 1 中依恋倾向为"焦虑型"，Layer 0 修正层写入：
```
「遇冷后」→ 实际应该：沉默1–2天后以中性话题重新开口，不回溯冲突
```

**场景：** 对手方突然冷淡，数字人（current_position = -70）进行回复

**验收标准：**
- [ ] 数字人行为遵循 Layer 0 的修正（沉默，不立刻追问）
- [ ] 不因 current_position 偏舔狗而覆盖修正层规则

---

### T5-2 数据矛盾时的并列记录

**操作：** 先喂入一段数据，分析得出"回避型"；再追加一段矛盾数据（表现为焦虑型）

**验收标准：**
- [ ] `merger.md` 执行后，profile 中两种信号并列记录，而非覆盖
- [ ] 标注时间差和"可能原因"字段

---

### T5-3 SWI 与 current_position 不绑定

**前置条件：** `current_position = +60`（海王区间），`current_swi = 30`（低）

**验收标准：**
- [ ] 对话行为体现海王特征（high position）
- [ ] 复盘摘要中 SWI 依然显示为低值，未被强制拉高
- [ ] 两个字段独立更新，无相互覆盖

---

### T5-4 空数据冷启动

**操作：** 跳过所有数据喂入，仅凭基础信息（性别/年龄/星座）完成初始化

**验收标准：**
- [ ] `persona.md` Layer 1–4 标注"待补充"，不生成伪造数据
- [ ] `self_worth_tracker.md` 的 `initial_swi` 标注"未评估"
- [ ] 系统可以进入对话（不因数据缺失报错），但行为较为模糊
