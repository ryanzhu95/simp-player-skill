# simp-player-skill — 产品需求文档 v1.0

---

## 一、产品概述

**simp-player-skill** 是一个关系动态模拟器。

建模对象是用户自己。用户将自己的行为模式提炼为数字人，选择以舔狗或海王模式运行，与不同对手方展开对话，数字人从每次对话中积累经验并持续演化。

---

## 二、核心概念

### 2.1 数字人（Digital Human）

用户自身的建模，是被持续塑造的对象。

- 由用户的基础信息 + 喂入的聊天记录/描述分析生成
- 人格用分层结构描述（Layer 0–4），Layer 0 为用户纠正的最高优先级修正层
- 所有对话共享同一个数字人，成长记录跨对话累积

### 2.2 模式谱（Mode Spectrum）

数字人在一条连续谱上运行，而非二元切换：

```
-100 ←————————————————————————————→ +100
纯舔狗                              纯海王
```

- **舔狗端**：焦虑依恋、低自我价值感、高追求强度、顺从冲突风格
- **海王端**：回避依恋、高自我呈现、撤退策略、多线操控
- 行为是 `current_position` 决定的混合态，位置越靠近某端，特征越纯粹
- 中间段（-20 ~ +20）呈现矛盾行为，是最接近真实人的区间

### 2.3 位置变化机制

```
target_position  → 数字人的内在意图，渗透进行为细节，但不自动推进位置
对话触发事件     → 真正驱动 current_position 移动的力量
非对称范围约束   → 每轮结束后 clamp 到 [target - range_regression, target + range_progression]
溢出记录         → 超出范围的部分记录为积压张力，带入下次对话开场状态
```

**范围默认值：**
- 初始模式 = 舔狗：`range_regression = 40`，`range_progression = 20`（更容易退回去）
- 初始模式 = 海王：`range_regression = 20`，`range_progression = 40`

### 2.4 自我价值感（SWI）

Self-Worth Index，范围 0–100，独立于模式谱追踪。

- 与 `current_position` 相关但不绑定：高 SWI 不等于海王，低 SWI 不等于舔狗
- 每轮对话后根据触发事件更新
- 长期趋势是观察数字人成长的核心指标之一

### 2.5 对手方（Opponent）

每次对话的另一方，画像从用户喂入的数据中分析生成，不使用预设模板。

- 每个对手方有独立的 `profile.md`，存储在 `users/sessions/{slug}/`
- 画像包含依恋倾向、冲突风格、爱之语、SWI 估值、追求节奏
- 包含"与数字人的关系化学"预测，基于双方画像推断典型动态

## 三、人格分析框架（Lenses）

分析视角库，被 `user_analyzer.md` 调用，适用于数字人自身和对手方。

| Lens | 分析内容 | 输出维度 |
|------|------|------|
| `attachment.md` | 依恋倾向 | 焦虑型 / 回避型 / 安全型 / 混乱型 |
| `conflict.md` | 冲突处理风格 | 顺从 / 回避 / 竞争 / 合作 / 妥协 |
| `love_language.md` | 爱之语 | 给出型 + 渴求型各一种 |
| `self_worth.md` | 自我价值感 | SWI 数值 + 波动特征 |
| `chase_rhythm.md` | 追求节奏 | 追求强度 + 撤退策略 + 热度维持 |

**分析原则：**
- 框架作为分析工具，标签值由数据得出，不由用户预设
- 有原文依据的结论引用原话；无依据的标注"（基于描述推断）"
- 矛盾信号如实并列，不强行统一

## 四、用户流程

```
用户触发 /simp-player
        ↓
[Step 1] 数字人初始化（首次运行）
  - 基础信息：性别 / 年龄 / 星座
  - 选择起始模式：舔狗 or 海王
  - 可选：喂入自身聊天记录 → user_analyzer → 生成 persona.md
        ↓
[Step 2] 对手方建档
  - 方式 A：粘贴聊天记录（推荐）
  - 方式 B：口头描述
  → user_analyzer → 生成 users/sessions/{slug}/profile.md
  → 展示画像摘要 + 关系化学预测，用户确认
        ↓
[Step 3] 对话
  - 数字人行为由 current_position + persona + 对手方画像共同决定
  - target_position 作为意图渗透进行为，但不自动推进位置
  - 积压张力影响开场状态
        ↓
[Step 4] 对话复盘（用户说"结束对话"时触发）
  - 识别触发事件，计算位移
  - 应用非对称范围约束，记录溢出
  - 更新 current_position / SWI / growth_log
  - 展示复盘摘要
        ↓
[持续] 成长循环
  - 追加对手方数据 → merger → 更新对手方画像
  - 更新自身数据   → merger → 更新 persona.md
  - 行为纠正       → correction_handler → 写入 Layer 0
  - 切换目标模式   → 更新 target_position，记录变更
```

## 六、成长机制详述

### 6.1 触发事件分类

| 事件 | 方向 | 参考位移 |
|------|------|------|
| 被明确拒绝 | ← 舔狗 | -10 ~ -20 |
| 被忽视/晾着 | ← 舔狗 | -5 ~ -15 |
| 被利用 | ← 舔狗 | -10 ~ -20 |
| 成功撤退（对方开始追） | → 海王 | +10 ~ +20 |
| 识别操控并未落入陷阱 | → 海王 | +10 ~ +15 |
| 表达边界 | → 海王 | +5 ~ +15 |
| 被深度认可 | → 海王 | +5 ~ +10 |
| 破防时刻（海王模式下情感失控） | ← 舔狗 | -15 ~ -30 |

### 6.2 溢出处理

位移结果超出 `[lower_bound, upper_bound]` 时：
- `current_position` clamp 到边界值
- 超出量记录为积压张力，写入 `growth_log.md`
- 积压张力在下次对话开场时影响数字人的情绪起点

### 6.3 SWI 与位置的关系

SWI 与 `current_position` 独立追踪，但存在关联：
- 长期处于 `current_position < -60` 会持续消耗 SWI
- 成功撤退/表达边界等事件同时推高 SWI 和 `current_position`
- SWI 的长期趋势比 `current_position` 更能反映数字人的真实成长

## 七、设计原则

1. **数据驱动，不预设结论** — 人格标签从数据中提取，用户无需自己贴标签
2. **成长优于还原** — 数字人不是静态快照，设计为在对话中持续演化
3. **溢出不丢弃** — 超出范围的情绪张力是有意义的信号，记录而非忽略
4. **模式是意图，不是进度条** — `target_position` 影响行为倾向，但不自动推进位置
5. **对话驱动成长** — 只有真实发生的对话事件才能改变数字人状态

## 八、未来迭代方向

- **range 动态收窄**：随着成长记录积累，`range_regression` 和 `range_progression` 逐渐向 target 收拢，模拟人格稳定化
- **多对手方交叉影响**：不同类型对手方的交替对话，对成长方向产生复合影响
- **关系阶段感知**：识别"暧昧期 / 热恋期 / 平稳期 / 冷战期"，调整触发事件的位移权重
- **导出功能**：将数字人的成长轨迹导出为可读报告
