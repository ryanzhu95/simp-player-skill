# 数据喂入入口（intake）

> 被 SKILL.md Step 1.1 和 Step 2 调用。
> 目标：引导用户提供原始数据，识别分析目标（数字人自身 or 对手方）。

---

## 调用时的上下文参数

```
target_type: digital_human | opponent   # 本次分析的是谁
slug: {标识符}                           # opponent 时必填
```

---

## 对话流程

### 若 target_type = digital_human

```
可以粘贴你自己参与的聊天记录，帮助我更准确地建模你的行为模式。

你是记录中的哪一方？（告诉我你用的名字或备注）

支持的格式：
- 直接粘贴文本聊天记录
- 截图（多张均可）
- 用几句话描述自己的典型行为也行

随时可以跳过，后续说"更新我的数据"可以补充。
```

### 若 target_type = opponent

```
需要先了解这次对话的对手方。

怎么称呼 TA？

可以：
A. 粘贴你们之间的聊天记录（分析更准确）
B. 用几句话描述这个人

支持格式同上。
```

根据称呼生成 `slug`：中文名转拼音小写（如"张三" → `zhangsan`），英文名转小写连字符（如"Alex Chen" → `alex-chen`），特殊字符替换为连字符。

---

## 数据质量提示

| 样本量 | 处理方式 |
|---|---|
| < 50 条对话 | 标注 ⚠️ 样本偏少，结论以描述为主，减少量化推断 |
| 50–200 条 | 正常分析，置信度标注"中" |
| > 200 条 | 正常分析，置信度标注"高" |
| 仅文字描述 | 全部结论标注"（基于描述推断）" |

---

## 完成后

将原始数据归档至：
- `digital_human/` 时 → `digital_human/raw_data/`
- `opponent` 时 → `users/sessions/{slug}/raw_data/`

然后调用 `prompts/user_analyzer.md`。
